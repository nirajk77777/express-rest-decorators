    **Chapter 2: The Big Break — Parameter Decorators → Method-Level Input** (LEAD CHAPTER — D-11 explicit)
    - Two clearly labeled fenced code blocks. Use these EXACT before/after snippets:

      Before (routing-controllers v0.11):

          @Get('/users/:id')
          async getUser(@Param('id') id: string, @Body() body: CreateUserDto) {
            return this.users.findById(id);
          }

      After (`@nirajk/express-controllers` v1):

          @Get('/users/:id', { params: { id: z.coerce.number() }, body: CreateUserSchema })
          async getUser({ params, body }: { params: { id: number }; body: CreateUser }) {
            return this.users.findById(params.id);
          }

    - One-paragraph rationale: the input declaration object lives at the method-decorator call site; type inference flows from schema → handler arg in one go; no per-arg decorator boilerplate.

    **Chapter 3: Breaking Changes Table** (single table, exactly two columns: `routing-controllers v0.11` | `@nirajk/express-controllers v1`)

    Rows MUST cover (8–10 rows is the target — no exhaustive RC-decorator-by-decorator mapping):

    | Feature                  | RC v0.11 | `@nirajk/express-controllers` v1 |
    |---|---|---|
    | Parameter decorators     | `@Param`, `@Body`, `@QueryParam`, `@HeaderParam`, `@CookieParam`, `@SessionParam`, `@UploadedFile` as ARG decorators | Method-level input declaration: `{ params, query, body, headers, cookies, session, files }` |
    | Koa support              | Yes (Express + Koa) | Removed — Express v5 only |
    | Express version          | v4 | v5.1+ peer (native async error propagation) |
    | Default validator        | class-validator + class-transformer | Standard Schema (Zod / Valibot / ArkType) |
    | DI hook                  | Global `useContainer(Container)` (TypeDI auto-imported in some setups) | Per-bootstrap `useContainer({ get: token => container.resolve(token) })` |
    | File uploads             | `@UploadedFile` arg decorator with implicit defaults | Method-level `{ files: { avatar: UploadedFile(...) } }` slot model with mandatory `limits` + `fileFilter` |
    | Cookie / session access  | `@CookieParam` / `@SessionParam` arg decorators | Method-level `{ cookies, session }` slots with optional schemas |
    | Path syntax              | path-to-regexp v0.x | path-to-regexp v8 (no unnamed regex groups; v4 patterns rejected at registration) |
    | Glob controller loading  | `controllers: ['src/**/*.ts']` via `require` | Same syntax via dynamic `import()` (tinyglobby + lazy peer) |
    | reflect-metadata         | Required for class-validator | Required by core (TS-emitted `design:paramtypes` reads) |

    **Chapter 4: Per-Feature Migration Recipes** (5–7 short subsections, ~1 page each)

    Subsections (use h3 `###`):
    1. Controllers + routing — `@Controller` / `@JsonController` / HTTP method decorators (largely unchanged signatures; note `routePrefix` continues to work)
    2. Input declaration — show params/query/body/headers slots; cross-link to chapter 2 for the big break
    3. Middleware, interceptors, authorization — `@UseBefore` / `@UseAfter` / `@Middleware` / `@Interceptor` / `@UseInterceptor` / `@Authorized` / `authorizationChecker` / `currentUserChecker` (largely RC-shaped; method-wins for `@Authorized`)
    4. File uploads — slot model with mandatory `limits`/`fileFilter`; multer is an optional peer
    5. Cookies / sessions — slot model; `cookie` / `express-session` as optional peers
    6. Dependency injection — `useContainer({ get: token => container.resolve(token) })` recipe (TypeDI / tsyringe / Awilix / InversifyJS)
    7. New on top — `getRequestContext()` (AsyncLocalStorage) + `printRoutes: true`

    **Chapter 5: What's Gone**
    - Koa support
    - Parameter decorators as the primary input mechanism (still importable as legacy decorators with arg metadata, but the docs-recommended path is method-level input)
    - class-validator / class-transformer as defaults (deferred to v1.x as an optional adapter; not in v1)
    - body-parser as a runtime dep (Express 5 ships `express.json()` / `express.urlencoded()`)

    **Chapter 6: What's New on Top**
    - `getRequestContext()` returning `{ req, res, requestId }` from anywhere in the call chain via AsyncLocalStorage (NEW-01 / NEW-02)
    - `printRoutes` boot option for dev-time route table dump (API-04)
    - Native Express v5 async error propagation — single library-installed error middleware (no per-handler try/catch)
