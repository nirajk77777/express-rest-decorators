    **Final boot strategy (authoritative — no alternatives):**

    `useExpressControllers` and `createExpressServer` BECOME `async`. Phase 2 had them synchronous; this plan changes the signature. Rationale: arity detection for user error middleware requires resolving the class to an instance at mount time (`isErrorMiddlewareInstance(instance)`), and `getContainer().get()` may return a Promise. There is no race-free way to register Express error middleware (4-arg) without resolving the instance first. Eager-at-boot resolution is the only correct path.

    Boot-time resolution rules (all happen inside the awaited `useExpressControllers` body, before any `app.use` for controller routers):
    1. **Global interceptors** (`options.interceptors ?? []`) — resolved ONCE via `await resolveInterceptorClasses(options.interceptors ?? [])`. The resulting `InterceptorInterface[]` is passed as `globalInterceptors` (already-resolved) into every `buildControllerRouter` call. Never re-resolved.
